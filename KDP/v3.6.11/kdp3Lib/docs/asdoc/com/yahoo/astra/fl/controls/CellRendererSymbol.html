A type of cell renderer that provides simple hooks for designers and quick developer mockups. Intended to be used as a Base Class for Library symbols in the Adobe Flash authoring tool. Extends MovieClip to allow timeline scripting.

See also

commitFunction provides a hook for updating children on stage every time the component redraws.

commitFunctionproperty
commitFunction:Function  [read-write]

A function hook used to allow designers or developers to pass data from the cell renderer to children placed on stage in the Flash authoring environment.

Note: This function, regardless of its origin is called from the scope of the cell renderer. The this keyword will always refer to the instance of the CellRendererSymbol.

Implementation
    public function get commitFunction():Function
    public function set commitFunction(value:Function):void

Example
The following code passes data to child components. This code should appear on the timeline in the cell renderer symbol. 
   this.commitFunction = function( data:Object ):void
   {
    // label is a TextField placed on stage
    this.label.text = data.label;
   
    // imageLoader is a UILoader placed on stage
    this.imageLoader.source = data.imageURL;
   }

Alternatively, you can access the listData object to get some automatically extracted data (from labelField, labelFunction, iconField, etc). The selected property is another important one, and you can access anything available on the renderer using this. 
   this.commitFunction = function( data:Object ):void
   {
    // label is a TextField placed on stage
    this.label.text = this.listData.label;
   
    // imageLoader is a UILoader placed on stage
    this.iconLoader.source = this.listData.source;
   
    //a rectangle in the background that is only visible when the item is selected
    this.selectedBackground.visible = this.selected;
   }

Warning: When commitFunction is defined as a timeline script, the CellRendererSymbol may be displayed before the frame scripts are run. TextFields with filler text or other objects with similar design placeholders may flicker for a very brief moment. This is generally not noticeable.
